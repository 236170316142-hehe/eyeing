================================================================================
  AUTOSTART METHODS (Windows - Multiple Fallbacks)
================================================================================

PRIMARY:
  Task Scheduler with ONLOGON trigger
  └─ Most reliable even if antivirus blocks startup folder

FALLBACK 1:
  VBS startup script in %APPDATA%\...\Startup
  └─ With 2-second boot delay + retry logic
  └─ Works with most shell integrations

FALLBACK 2:
  Registry: HKCU\Software\Microsoft\Windows\CurrentVersion\Run
  └─ User-level registry entry
  └─ Survives if admin policies block startup folder

FALLBACK 3:
  Registry: HKLM\Software\Microsoft\Windows\CurrentVersion\Run
  └─ System-wide entry (requires admin to set)
  └─ Works even if user registry modified

Result: If ANY method is active, monitor will restart on system boot

If verify_autostart.py reports FAIL:

  → Last resort: Manual setup via "shell:startup" folder

  DEPLOYMENT TO OTHER PCs (Portable Package)

1. PREPARE SOURCE PC:
   mkdir deployment_package
   copy *.py deployment_package/
   copy *.bat deployment_package/
   copy *.txt deployment_package/
   copy tesseract-ocr-w64-setup-*.exe deployment_package/ (optional)
   
2. ZIP AND TRANSFER:
   7z a employee-monitor-package.zip deployment_package/

   Restart computer
